# 项目文档索引

> **核心目的**：让 AI 和开发者先通过文档理解项目，再按任务定向阅读源码，避免无目的全局扫代码。

## 快速导航

| 文档 | 内容 | 适用场景 |
|------|------|----------|
| [项目概览](overview.md) | 技术栈、目录结构、核心概念 | 首次接触项目 |
| [架构与数据流](architecture.md) | 页面渲染流程、组件关系、数据流向 | 理解整体运作机制 |
| [组件清单](components.md) | 所有组件的职责、依赖、所在文件 | 修改或新增组件时查阅 |
| [API 路由](api.md) | 后端接口定义、环境变量、第三方集成 | 调试接口或新增 API 时查阅 |
| [配置与环境](configuration.md) | 配置文件说明、环境变量、Docker 部署 | 环境搭建和部署时查阅 |
| [任务索引](task-index.md) | "我想做 X → 去看哪些文件" 的映射表 | 带着具体任务来时优先阅读 |

## 阅读建议

1. **第一次来**：先读 [项目概览](overview.md)，建立全局认知
2. **带着任务来**：直接查 [任务索引](task-index.md)，定位到具体文件
3. **需要深入**：根据任务索引指引，阅读对应的架构或组件文档
4. **开始编码**：按 [CLAUDE.md](../CLAUDE.md) 中的第一性原理规范执行
